# Refactor Blueprint

> One-shot DAG for code refactoring with complexity gate and circular dependency check.

## DAG

```
git-branch
  → system-supervisor-quick-audit (deterministic)
  → turing-complexity-gate (deterministic)
  → implement-refactor(1)
  → lint
  → [fix-lint(1)]
  → type-check
  → test
  → [fix-ci(2)]
  → circular-deps-check (deterministic)
  → git-commit
  → git-push
  → create-pr
```

## System Supervisor Quick Audit (Deterministic)

Before any agentic work, read these files and grep for red flags:

1. `cat .claude/memory/current-state.md` — check for existing escalations
2. `grep -r "TODO|FIXME|HACK" {target-file}` — catalogue technical debt
3. `grep -r "console.log|print(" {target-file}` — find debug statements

Output a brief audit summary (no LLM — just counts and line numbers).

## Turing Complexity Gate (Deterministic)

Before `implement-refactor`, check the target file for complexity signals:

1. Nested loops (grep `for.*for|while.*while` — O(n^2) risk)
2. Deeply nested conditionals (more than 3 levels — grep `if.*if.*if`)
3. Functions over 50 lines (wc -l on each function)

If O(n^2) patterns detected → add warning to PR: "Council of Logic flag: potential O(n^2) pattern detected at line {n}. Reviewer should verify complexity."

## Pre-Hydration Context (loaded before `implement-refactor` node)

Always loaded:

- `.claude/memory/CONSTITUTION.md`
- `.claude/memory/current-state.md`

General domain:

- The specific file(s) being refactored
- Direct imports (one level only)
- Relevant test files

## Escalation Conditions

- `system-supervisor-quick-audit` finds existing escalation in current-state.md → resolve that first
- `implement-refactor` changes more than 5 files → escalate for human review (scope creep risk)
- `circular-deps-check` finds new circular dependencies → escalate (do not merge)
- `fix-ci` reaches 2 rounds → escalate
