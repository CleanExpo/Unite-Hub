# Migration Blueprint

> One-shot DAG for database and code migrations. HIGH RISK — execution-guardian gate is mandatory.

## DAG

```
execution-guardian-check (deterministic risk gate)
  → [HIGH risk → BLUEPRINT_ESCALATION immediately]
  → git-branch
  → backup-check (deterministic)
  → implement-migration(1)
  → test
  → [fix-ci(1, reduced cap)]
  → type-check
  → lint
  → git-commit
  → git-push
  → create-pr
```

## Execution Guardian Risk Gate (Deterministic)

Map task keywords to risk level — no LLM involved:

| Keyword Pattern                                       | Risk Level | Action                     |
| ----------------------------------------------------- | ---------- | -------------------------- |
| `drop column\|drop table\|delete migration\|truncate` | HIGH       | Immediate escalation       |
| `auth\|jwt\|oauth\|rbac\|permission\|role`            | HIGH       | Immediate escalation       |
| `production\|prod env\|live database`                 | HIGH       | Immediate escalation       |
| `add column\|new table\|new migration\|alter column`  | MEDIUM     | Proceed with warning in PR |
| `dependency upgrade\|package version`                 | MEDIUM     | Proceed with warning in PR |
| `rename\|refactor\|restructure`                       | LOW        | Proceed normally           |

**HIGH risk → output BLUEPRINT_ESCALATION immediately, do not proceed.**

## Backup Check (Deterministic)

Before `implement-migration`:

1. Verify the migration SQL is reversible (include a rollback comment or separate down migration)
2. Verify git is clean (no uncommitted work to lose)
3. Verify Supabase project is accessible (local or remote)

## Pre-Hydration Context (loaded before `implement-migration` node)

Always loaded:

- `.claude/memory/CONSTITUTION.md`
- `.claude/memory/current-state.md`

Database domain:

- `supabase/migrations/` (existing migrations for context)
- `supabase/config.toml`
- `src/lib/supabase/server.ts`

## Iteration Caps (Reduced for Migrations — Higher Risk)

| Node                  | Cap                         |
| --------------------- | --------------------------- |
| `implement-migration` | 1                           |
| `fix-ci`              | 1 (reduced from standard 2) |
| **Total**             | **2**                       |

## Escalation Conditions

- `execution-guardian-check` returns HIGH risk → immediate escalation (no git branch created)
- `backup-check` fails (no rollback SQL in migration) → escalation
- `implement-migration` produces no migration file → escalation
- `fix-ci` reaches 1 round with failures → escalation (reduced tolerance)
